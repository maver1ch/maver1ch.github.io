## Problem

HTI Group needed a single chatbot to unify fragmented HR institutional knowledge spread across **Excel, Docs, Markdown, PowerPoint, PNG, and CSV** sources. Off-the-shelf RAG broke down on complex policy queries that crossed multiple sub-domains: leave requests, payroll, benefits, and tooling. The system also had to evolve from a quick proof-of-concept into a production-grade AWS deployment without rewriting the agent layer.

## Key Decisions

<DecisionBlock
  decision="LangGraph + ReAct multi-agent over a single mega-prompt"
  why="Separation of concerns. Each HR sub-domain (policies, leave, payroll) becomes its own agent with bounded context. Easier to test, eval, and route. Cheap to add a new domain — drop in a new agent and update the router."
  tradeoff="More moving parts. Requires distributed tracing (solved with Langfuse) to debug routing decisions across agents."
/>

<DecisionBlock
  decision="Hierarchical Semantic Chunking with Auto Merge + LLM Contextualization"
  why="Naive fixed-size chunking lost the relationship between sections. Hierarchical chunks preserve parent–child context; LLM contextualization injects a one-line summary into each child chunk so retrieval hits land with their original meaning intact."
  tradeoff="Higher offline ingestion cost. Acceptable because HR documents update slowly and retrieval quality is the bottleneck, not ingestion throughput."
/>

<DecisionBlock
  decision="Hybrid retrieval: Vector + Full-text + Graph + Pandas"
  why="HR data is heterogeneous. Vector search wins on semantic queries; full-text on policy IDs and exact phrases; graph on org-structure questions; Pandas on tabular data like leave balances. A single retriever could not cover all four, so we route per query type."
  tradeoff="Three retrievers to maintain. Mitigated by keeping each one simple and orchestrated by the LangGraph router."
/>

<DecisionBlock
  decision="AWS CDK in TypeScript for infrastructure"
  why="The agent layer was Python, the infra layer needed strong typing for the team's React/TS background. CDK gave us type-safe IaC with reusable constructs for Lambda + ECS + observability stacks."
  tradeoff="Two languages in one repo. Worth it for repeatable, peer-reviewed deploys instead of clicking through the AWS console."
/>

<DecisionBlock
  decision="Distributed Langfuse observability from Day 1"
  why="Multi-agent systems fail in ways that single LLM calls do not — wrong routing, lost context, hallucinated tool calls. Langfuse traces every step so we can attribute failures to the correct sub-agent and ship fixes."
  tradeoff="Added latency and storage cost. Both are negligible compared to debugging blind in production."
/>

## Outcome

- **Unified institutional knowledge** across 6+ document formats behind one chatbot interface.
- **Production-ready on AWS** with two deployment topologies (Lambda + ECS Fargate, plus a serverless ingestion alternative).
- **Hallucination-free outputs** via Langfuse-traced agent trajectories and per-step evaluation.
- **Reusable agent and infra patterns** that informed downstream projects at FPT Software.
- **Tool calling to external services** (Discord, Google Calendar) integrated cleanly through the agent framework.
